React 101

Posted on 2015-11-29

Recently, I am working on a freelancing project to build a SPA. Since I have learnt React for a bit while, I decided to put it in use this time. For me, the two most significant benefits that React brings to us is (1) its performance – due to React’s virtual DOM and DOM diff mechanisms; and, (2) its way to wrap and abstract the (major) building blocks of a web app into components.

For those who have yet to hear/learn React, here is a very quick and brief instruction/tutorial to guide you onto the track.

Component Life Cycle

The Life Cycle of each React Component contains three stages:

  • Mounting: Attaching to DOM.
  • Updating: Being rendered.
  • Unmounting: Removing from DOM.

Each of these three stages (excepts unmount) is assigned two pre- (will) and post-stage (did) methods:

  • componentWillMount()
  • componentDidMount()
  • componentWillUpdate(object nextProps, object nextState)
  • componentDidUpdate(object prevProps, object prevState)
  • componentWillUnmount()

There are two additional methods:

  • componentWillReceiveProps(object nextProps): Component about to receive props.
  • shouldComponentUpdate(object nextProps, object nextState)：When detecting if need to re-rendered.
